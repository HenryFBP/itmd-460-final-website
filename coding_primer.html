Java crash course

What is Java?

Java is a very popular programming language used in quite a lot of places.

Java has quite a few nice features, such as:

  • Portability due to the JVM interpreting Java bytecode, and the JVM which is on nearly every platform (Ever notice you don't need to recompile Java programs for different platforms, OSes, or processors?)

    • Note: The thing that is different across platforms is the JVM itself. This step always has to exist, as long as processors and OSes contain fundamental differences from eachother.

  • Concurrency (Multiple things at once, using synchronized

  • Structured programming (for loops, if statements)

  • Functional programming (only Java 8 and up, lambdas)

  • Object Orientation, i.e. inheritance child/parent relationships

  • Abstraction (interfaces, abstract classes)

To name a few.

Also, memory management is done for you via a garbage collection system. No malloc() or free() here.

It is also statically typed, meaning that the types of all variables are inferred upon compilation.

To sum up Java’s static typing, the code below will produce a compilation error:

String x = "Hello";
x = 3; //I'm a bad line of code ;)

However, you can assign types to variables freely in other languages. This feature is both a pro and a con, as most features end up being.

What does the Java language look like?

The Java language looks somewhat C-like, as it is whitespace-insensitive, and enjoys semicolons and curly braces.

Below is the definition of very simple Java function that takes a number as an argument, and returns a number back to you.

If the number is over 20, you get returned the number you put in, plus 200. Otherwise, you will get 20.

public Integer stuff(Integer argument) {

    if (argument >= 20) {
        return (argument + 200);
    }

    String msg = "Hello there!";

    if (msg.contains("the")) {
        return 20;
    } else {
        return -1;
    }
}

If you prefer mathematical notation, the following formula describes it too:

$$ stuff(x) = \left\{\begin{matrix} x + 200 & x \ge 20 \\ 20 & x < 20 \end{matrix}\right. $$

All the words in blue are reserved syntax elements that do specific things in Java. You can generally name your variables anything you want.

The 'types' in this code snippet are 'String' and 'Integer'.

Objects can have 'functions' that operate on their data, and these functions are called 'methods'.

Relating this code to real life, we could say that a car has a 'drive' method, and you have an 'eat' method.

I used the String.contains(…) method to demonstrate this principle.

If we wanted to print the value 220 to the console, we could write the following:

System.out.println(stuff(20));

You can see that invoking the function is as easy as putting its name, plus a value wrapped inside it.

We say we have 'called' the stuff function with the argument 20.

XML crash course

Before I describe activity_main.xml, one of our project's main files, I have to describe what XML is to you.

So, XML. It stands for eXtensible Markup Language.

This is a markup language (as opposed to a programming language) which descends from the same standard that HTML, HyperText Markup Language, does. Their common parent is a standard called SGML, which descended from an ancient standard called GML, and its ancestor, Rock-Based Flint Inscription Markup Language (RBFIML)[1], also known as flint knapping.

On a high level, it is a way to annotate a document or piece of data in such a way that the annotations themselves are able to be distinguished from the text.

Simply put, it is a system for combining data, metadata, and structure in a way that allows for them to be separated again.

Let's look at a very simple snippet of XML that describes a fridge:

<?xml version="1.0" encoding="utf-8" ?>
<fridge>
    <shelf number="0"> 
        <eggs amount="3" /> 
        <cheese attributes="moldy" smell="smelly"> 
            <amount> 
                1 lb
            </amount>
        </cheese>

    </shelf>
    <shelf number="1" /> 

    <shelf number="2">
        
    </shelf>
</fridge>

This XML snippet describes a fridge with three shelves, and the top shelf has 3 eggs and 1lb of yucky cheese.

Some important notes here are:

  • Every opening tag must close.

  • Tags can have 0 or more attribute-value pairs, as you can see with <cheese>.

    • All attribute values must be enclosed with double quotes.

  • All tag and attribute names can NOT have spaces or other crazy stuff in them.

  • A tag can have 0 or more children, as you can see with <fridge>.

  • If a tag has no children, it can self-close, as you can see with <shelf number="1" />

  • There can only be one 'root' tag.
