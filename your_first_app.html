Your First App

I have also made a video version of this page, and it is shown below.

This is the same as the first one.

Here is a link to it on YouTube.

  1. Click on "Start a new Android Studio Project".

    1. Unless you want Kotlin support (A language that compiles into Java bytecode), uncheck "Include Kotlin Support".

      If you don't know Kotlin, this step can create moderate problems for you.

  2. Click on Next. Now you can select API versions.

    Then, select which API versions you wish to target. The default, 16, is fine.

    Why is this a step?

    Well, with every Android update, the things you can do (and can NOT do) change!

    Certain libraries become deprecated in favor of new ones, and new libraries are made.

    A lower API version means your app will be able to run on more devices, but must use older libraries.

    A higher API version means your app will be able to run on less devices, but can use new features.

  3. Click on Next. This is a handy screen that lets you select what your first app's screen will look like.

    For us, let's do 'Empty Activity'.

    Click on Finish, and wait.

  4. Before we edit anything, I want to just run our app.

    1. However, before we do this, we need to set up an "Android Virtual Device".

      This is so that we can run it without needing an Android phone.

      I won't cover that here, as I don't have one with a sufficient OS version.

      If you want to know how to do this, Google "Android Debugging Bridge".

  5. Go to Tools > AVD Manager > Create Virtual Device

    1. You can select the hardware profile of the device.

      This will determine its screen size, (lack of) cameras, and other hardware stuff.

      This doesn't really matter, as most Android apps work on all scales, if made properly.

    2. Let's use a Nexus 5X because 'Why Not™'?

    3. Next, you'll need to select a 'system image'. This is where those processor features are actually important.

      1. If the architecture of the system image is x86 or x86_64, then you need VT-x enabled on your processor to run it. Virtualization is a prerequisite for VT-x.

      2. If you DON'T have virtualization or VT-x, you can use an arm64 or armeabi image, but they will be very slow.

    4. I will use API 28. Also, more downloading.

  6. Once the download is finished, select the downloaded system image and click 'Next', and then click 'Finish'.

  7. Click the green 'Run' button in the upper-right, or do 'Shift-F10'.

    YAY! It works!
